# Writing Docs - Document Index

Supporting documentation for the writing-docs skill. Each entry describes what
the document covers and when to read it.

## Documents

- **[progressive-disclosure.md](progressive-disclosure.md)**: Patterns for
  splitting documentation across multiple files so agents load only what's
  needed for the current task. Read when organizing multi-file documentation,
  skills, or reference material that exceeds ~300 lines.

- **[writing-descriptions.md](writing-descriptions.md)**: How to write effective
  descriptions for document index entries, inline links, and skill YAML
  `description` fields. Covers the "should I read this?" test, good/bad
  examples, and key term selection. Read when writing index entries, skill
  frontmatter, or any reference link.

- **[document-hierarchy.md](document-hierarchy.md)**: How to structure document
  hierarchies using `docs/index.md` as the root, with nesting rules,
  sub-index patterns, and the golden rule enforcement checklist. Read when
  setting up new project documentation or reorganizing an existing doc tree.
